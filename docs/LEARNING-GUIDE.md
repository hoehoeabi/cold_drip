# 📘 리액트 입문자를 위한 프로젝트 가이드

이 문서는 리액트를 처음 접하는 분들이 이 프로젝트의 구조와 문법을 쉽게 이해할 수 있도록 정리한 가이드입니다.

---

## 1. 🏗️ 프로젝트 아키텍처 (구조)

이 프로젝트는 **"관심사 분리(SoC)"** 원칙에 따라 폴더가 나누어져 있습니다.

- **`src/shared` (공통)**: 앱 전체에서 공통으로 쓰는 것들 (로그인 정보 바구니, 버튼/입력창 부품).
- **`src/features` (기능)**: 특정 기능에 집중된 코드들 (게시글 작성 로직, 댓글 로직 등).
- **`src/pages` (페이지)**: 여러 기능을 조합해서 만든 하나의 화면 (메인 화면, 마이페이지).
- **`src/App.jsx` (네비게이터)**: 어떤 주소(/, /mypage)에서 어떤 페이지를 보여줄지 결정하는 설계도.

---

## 2. ⚛️ 이 프로젝트에 사용된 핵심 리액트 문법

### ① useState (상태 관리)
화면에서 변해야 하는 모든 데이터는 `useState`에 담습니다.
- **사용 예**: 입력창의 글자, 서버에서 가져온 게시글 목록 등.
- **원리**: 우리가 변수값을 바꾸면 리액트가 그걸 감지하고 **"화면을 다시 그려줍니다."**

### ② useEffect (실행 시점 관리)
"언제 이 코드를 실행할 것인가?"를 결정합니다.
- **사용 예**: 페이지가 처음 켜졌을 때 DB에서 데이터 가져오기, 특정 변수가 바뀔 때만 코드 실행하기.

### ③ useContext (공용 바구니)
로그인 정보처럼 모든 곳에서 필요한 데이터는 일일이 전달하지 않고 **Context(바구니)**에 넣어둡니다.
- **해결한 문제**: 부모 -> 자식 -> 손자 -> 증손자로 데이터를 전달하는 번거로움(Prop Drilling)을 없앴습니다.
- **적용 사례**: 
    - `AuthContext`: 로그인 유저 정보 관리
    - `MessageContext`: 알림 메시지 관리
    - `ThemeContext`: 다크모드/라이트모드 상태 관리

### ④ React Router (주소창 관리)
웹사이트 주소에 따라 화면을 바꿔주는 도구입니다.
- `<Link>`: 새로고침 없이 부드럽게 주소를 바꿉니다.
- `useNavigate`: 코드 안에서 (예: 로그인 성공 시) 페이지를 강제로 이동시킵니다.

---

## 3. 🛠️ 적용된 기술 스택 설명

### ① Supabase (슈파베이스)
백엔드 서버를 직접 만들지 않고 빌려 쓰는 서비스입니다.
- **Auth**: 회원가입, 로그인 담당.
- **Database**: 게시글, 댓글 저장 담당 (PostgreSQL).
- **Storage**: 사진 파일 저장 담당.

### ② Tailwind CSS (테일윈드)
CSS 파일을 따로 만들지 않고 HTML 태그 안에 `className="bg-blue-500"` 처럼 적어서 디자인하는 방식입니다.
- **장점**: 디자인 수정이 매우 빠르고, 클래스 이름을 고민할 필요가 없습니다.

---

## 4. 🚀 리팩토링의 역사 (우리가 한 일)

1. **초기 단계**: `App.jsx` 한 파일에 수백 줄의 코드가 다 들어있었습니다. (작업하기 매우 힘든 상태)
2. **모듈화**: 코드를 성격에 맞게 `features`, `pages`로 쪼개어 가독성을 높였습니다.
3. **Context 도입**: 공통 데이터(로그인, 알림)를 전역 바구니에 담아 관리를 편하게 했습니다.
4. **Tailwind 전환**: 복잡한 CSS 파일을 제거하고 현대적인 방식으로 디자인을 통일했습니다.
